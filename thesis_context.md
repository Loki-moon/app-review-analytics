초록

디지털 플랫폼 환경에서 기업은 다양한 경로로 고객의 반응을 모니터링하는데 그 중 일부는 앱 마켓 리뷰 데이터를 활용하여 서비스 문제를 진단하고 개선안을 도출한다. 그러나 대부분의 분석은 자사 앱에 한정되어 이루어지며, 동일 카테고리 내 경쟁 앱과의 상대적 기능 경쟁력을 정량적으로 판단하기에는 한계가 있다. 특히 내부 로그 데이터는 기밀성과 접근 제약으로 인해 외부 비교 분석에 활용되기 어렵기 때문에, 기능 수준의 경쟁력을 객관적으로 비교할 수 있는 분석 틀이 요구된다. 이에 본 연구는 공개 데이터인 앱 마켓 리뷰를 활용하여 기능 수준의 상대적 경쟁력을 정량적으로 측정하는 분석 프레임워크를 제안한다.
본 연구는 앱 마켓 리뷰 텍스트에 포함된 기능 관련 키워드를 독립 변수로 설정하고, 평점(4~5점=긍정, 1~2점=부정)을 이분화한 종속 변수로 설정하여 로지스틱 회귀모형을 추정하였다. 이를 통해 각 키워드가 긍정 평가 확률에 미치는 조건부 영향력을 오즈비(Odds Ratio)로 정령화하고, 동일 기능 키워드에 대한 오즈비를 앱 간 비교함으로써 기능 경쟁력의 상대적 격차(OR)를 산출하였다. 나아가 OR과 자사 취약도를 결합한 우선순위 점수 규칙을 제시하고, 통제 변수 설정과 강건성 검증 그리고 민감도 분석을 통해 추정 결과의 신뢰성을 확인하였다.
사례 연구 대상으로는 간편결제 앱인 네이버페이와 카카오페이를 선정하였다. Google Play Store에서 최근 1년간 수집한 리뷰를 전처리(정규화, 형태소 기반 토큰화, 불용어 제거)한 후 분석한 결과, 동일 기능이라도 앱에 따라 긍정과 부정 평가 구조가 상이하게 나타나는 키워드가 존재함을 확인하였다. 또한 오즈비 격차와 자사 취약도를 결합한 우선순위 점수를 통해 기능 개선의 전략적 방향을 도출할 수 있음을 보였다.
본 연구는 리뷰 분석을 단일 서비스 진단에서 경쟁 비교 구조로 확장하였다는 점에서 학문적 의의를 가지며, 해석 가능한 통계 모형을 활용하여 기능 단위 영향력을 의사결정 규칙으로 전환하는 방법론을 제시한다. 또한 내부 데이터 접근이 제한된 상황에서도 공개 리뷰 데이터만으로 기능 경쟁력을 비교할 수 있는 실무적 적용 가능성을 제시한다. 제안된 프레임워크는 간편결제 도메인 외에도 커머스, 배달, OTT, 헬스케어 등 앱 마켓 리뷰가 존재하는 다양한 디지털 서비스 카테고리로 확장 적용될 수 있다.

제 1 장 서론


1. 연구의 필요성과 목적
디지털 플랫폼 환경에서 앱 서비스는 지속적인 기능 확장과 사용자 경험(UX) 개선을 통해 경쟁 우위를 창출하고 유지한다. 그러나 서비스 기획자는 항상 한정된 개발·디자인 자원 내에서 “어떤 기능을 우선적으로 개선할 것인가”라는 전략적 선택 문제에 직면한다. 이는 단순한 운영상의 문제가 아니라, 제한된 자원을 효율적으로 배분하여 사용자 만족도와 핵심 성과 지표(KPI)를 극대화해야 하는 자원 배분 최적화 문제에 해당한다. 특히 기능 복잡성이 증가하고 서비스 간 경쟁이 심화될수록, 기능 개선의 우선순위를 객관적 근거에 기반하여 설정하는 문제는 더욱 중요해진다.
일반적으로 이러한 의사결정은 내부 행동 데이터(이벤트 로그, 전환 퍼널, 리텐션, 체류 시간 등)에 기반하여 이루어진다. 내부 데이터는 사용자 행동을 정량적으로 설명할 수 있는 강력한 근거를 제공하며 조직 내부의 운영 최적화와 성과 관리에는 유용하다. 그러나 내부 데이터는 기업 기밀 및 개인정보 보호 이슈로 인해 외부 공개가 제한되며, 경쟁 서비스 간 비교 분석이나 학술 연구 차원에서 구조적으로 접근이 어렵다. 결과적으로 내부 데이터 기반 의사결정은 조직 내부 성과 최적화에는 적합하나, 시장 전체 맥락에서의 상대적 기능 경쟁력 판단에는 본질적인 제약을 가진다.
이에 비해 앱 마켓 리뷰 데이터는 사용자가 자발적으로 생성한 공개 텍스트 데이터로서 서비스 품질, 오류, 정책, 사용자 경험, 고객 대응 등 전반적인 사용 경험에 대한 인식을 반영한다. 특히 동일 카테고리 내 경쟁 앱의 리뷰 역시 공개되어 있다는 점은 기능 수준에서의 시장 반응을 비교 가능하게 한다는 점에서 중요한 분석 자원으로 기능한다.
그럼에도 불구하고 실무 현장에서의 리뷰 분석은 대체로 자사 앱 단일 분석에 한정되는 경향을 보인다. 이는 특정 서비스의 문제 요인을 도출하는 데에는 기여할 수 있으나, 동일 기능이 경쟁 서비스에서 어떻게 평가되는지, 즉 기능의 상대적 경쟁력을 정량적으로 판단하는 데에는 한계를 가진다. 동일 기능의 불만이 산업 전반의 공통 현상인지, 특정 서비스의 구조적 취약점인지를 식별하기 어렵고, 기능 개선의 시급성이 내부 기준에 의해 결정될 위험이 존재한다.
기존 연구 또한 단일 서비스의 만족도 결정 요인을 규명하거나 감성 분석을 통해 리뷰의 긍 · 부정 키워드를 분류하는 데 집중해 왔다. 그러나 동일 기능을 기준으로 복수 앱 간 비교를 수행하고, 그 결과를 기능 우선순위 결정 체계로 연결하는 통합적 분석 방법론은 충분히 제시되지 않았다.
이에 본 연구는 공개 데이터인 앱 마켓 리뷰를 활용하여 기능 수준의 상대적 경쟁력을 정량적으로 측정하는 분석 프레임워크를 제안한다. 구체적으로 기능 관련 키워드를 독립 변수로 설정하고, 사용자 평점을 종속 변수로 하는 로지스틱 회귀모형을 통해 각 키워드가 긍·부정 평점 발생 확률에 미치는 조건부 영향력을 추정한다. 나아가 추정된 계수를 오즈비(Odds Ratio)로 변환하여 해석 가능한 지표로 정량화하고, 동일 기능 키워드에 대한 앱 간 비교를 통해 기능 경쟁력을 정의하며, 이를 기능 개선 우선순위 결정 규칙으로 전환하고자 한다.
본 연구는 기능 경쟁력을 평균 평점이나 사용자 수와 같은 거시적 지표가 아닌, 동일 기능이 언급되었을 때 긍·부정 평가 확률이 어떻게 변화하는지에 대한 조건부 영향력의 차이로 개념화한다. 이는 기능 단위의 미시적 경쟁력을 분석 대상으로 삼는다는 점에서 기존 연구와 구별된다.
이러한 개념적 정의를 실증적으로 검증하기 위하여, 다음 절에서는 연구 문제를 설정하고 이에 대응하는 가설과 분석 모형을 구체화한다.

2. 연구 가설 및 실험 모형
본 연구는 앱 마켓 리뷰라는 공개 텍스트 데이터를 활용하여 기능 수준의 상대적 경쟁력을 정량적으로 추정하고, 이를 기능 개선 의사결정 체계로 전환하는 분석 모형을 제안한다. 기존 연구가 단일 서비스 내부의 만족 요인을 규명하는 데 주력해 왔다면, 본 연구는 동일 기능이 언급되었을 때 긍·부정 평가 확률이 서비스 간 어떻게 달라지는지를 비교함으로써 기능 단위 경쟁력을 조건부 영향력의 차이로 개념화한다.
이를 위해 실증 분석 대상은 간편결제 카테고리의 대표 서비스인 카카오페이와 네이버페이로 설정하였다. 두 서비스는 결제, 송금, 포인트, 오프라인 결제, 생체 인증 등 핵심 기능군이 구조적으로 유사하며, 동일 기능을 기준으로 한 비교가 가능한 경쟁 구도를 형성하고 있다. 또한 간편결제 도메인은 인증 실패, 결제 오류, 네트워크 지연, 업데이트 이후 오류 발생 등 반복적으로 관찰되는 기능 관련 불만 유형이 존재한다는 점에서 기능 단위 영향력을 검증하기에 적합한 환경을 제공한다.
따라서 본 연구는 리뷰 텍스트에 나타난 기능 키워드가 평점에 미치는 영향을 추정하고, 동일 기능에 대한 서비스 간 영향력 차이를 통계적으로 검증함으로써, 기능 수준의 상대적 경쟁력을 실증적으로 도출하고자 한다.

본 프레임워크는 간편결제 외 다른 디지털 서비스 카테고리로 확장 가능한가?


2.2 연구 가설 (Hypotheses)
본 연구는 기능 경쟁력을 동일한 기능이 리뷰에 언급되었을 때의 긍·부정 평가 확률 변화로 정의한다. 리뷰에 나타난 기능 키워드는 사용자 경험에 대한 인식적 평가를 반영하며, 이는 평점에 체계적인 영향을 미칠 것으로 예상된다. 또한 동일 기능이라 하더라도 서비스 간 구현 방식과 운영 안정성의 차이에 따라 영향력의 크기는 상이하게 나타날 수 있다. 이러한 논리에 근거하여 다음과 같은 가설을 설정한다.
<표 1.2> 연구 가설
H.1
기능 관련 키워드의 등장 여부는 긍·부정 평점 발생 확률에 통계적으로 유의한 영향을 미친다.
H.2
동일 기능 키워드의 영향력(오즈비)은 앱 간 통계적으로 유의한 차이를 보인다.
H.3
서비스 간 영향력과 자사 취약도를 결합한 지표는 기능 개선 우선순위 설정에 유효한 기준이 된다.


2.3 연구 흐름 
본 연구는 “리뷰 데이터 수집 → 텍스트 전처리 및 기능 키워드 구성 → 평점 이분화 → 로지스틱 회귀 기반 영향력 추정 → 오즈비 산출 및 앱 간 격차 비교 → 기능 개선 우선순위 도출”의 흐름으로 구성된다.
즉, 리뷰에서 특정 기능이 언급되었을 때 긍정(또는 부정) 평가로 이어질 가능성이 어떻게 달라지는지를 추정하고, 동일 기능에 대한 영향력 차이를 서비스 간 비교함으로써 기능 경쟁력을 정량화한다. 이때의 상세 변수 정의, 모형 설정, 통제변수 구성, 격차 검정 절차 및 우선순위 산출 규칙은 “제 3 장”에서 구체적으로 제시한다.

3. 연구 시사점과 한계
본 연구는 공개 앱 마켓 리뷰 데이터를 활용하여 기능 단위 경쟁력을 정량적으로 비교하고, 이를 기능 개선 의사결정 규칙으로 전환할 수 있는 분석 틀을 제시하였다. 기존 연구가 단일 서비스 내부의 만족 요인 규명에 초점을 두었다면, 본 연구는 동일 기능이 언급되었을 때의 긍·부정 평가 확률을 조건부로 추정하고 이를 서비스 간 비교함으로써 기능 경쟁력을 미시적 영향력의 차이로 개념화하였다. 이는 평균 평점이나 사용자 규모와 같은 거시적 지표와 구별되는 기능 단위 분석 관점을 제시하였다는 점에서 의의를 가진다.
또한 로지스틱 회귀와 오즈비를 활용하여 기능 영향력 추정, 앱 간 격차 검정, 우선순위 도출로 이어지는 해석 가능한 분석 구조를 설계하였다. 이를 통해 리뷰 분석을 단순한 문제 진단을 넘어 경쟁 비교 기반의 전략적 의사결정 지원 체계로 확장하였다. 아울러 내부 데이터 접근이 제한된 상황에서도 공개 리뷰 데이터만으로 기능 개선 우선순위를 도출할 수 있음을 보였으며, 제안된 프레임워크는 동일 기능 비교가 가능한 다양한 디지털 서비스 영역으로 확장 적용이 가능하다.
그러나 본 연구는 몇 가지 한계를 가진다. 리뷰 데이터는 자발적 참여 기반으로 생성되므로 표본 선택 편향이 존재할 수 있으며, 특정 업데이트나 장애 발생 직후의 시점 효과를 완전히 통제하기 어렵다. 또한 리뷰는 사용자 인식을 반영하는 데이터로서 실제 행동 지표와 직접적으로 동일시하기에는 한계가 있다. 기능 키워드 사전 구성 과정에서도 연구자의 해석이 개입될 가능성이 존재하며, 실증 분석 대상이 간편결제 카테고리에 한정되어 있어 결과의 일반화에는 신중한 접근이 필요하다.
<표 1.3> 연구 한계
한계.1
리뷰 데이터는 자발적 참여 기반으로 생성되므로 표본 선택 편향이 존재할 수 있다.
한계.2
특정 업데이트나 장애 발생 직후의 시점 효과를 완전히 통제하기 어렵다.
한계.3
리뷰는 사용자 인식을 반영하는 데이터로, 실제 행동 지표와 직접적으로 동일시할 수 없다.
한계.4
기능 키워드 사전 구성 과정에서 연구자의 해석 개입 가능성이 존재한다.
한계.5
실증 대상이 간편결제 카테고리에 한정되어 있어 일반화에는 신중한 해석이 필요하다.
한계.6
기능 키워드의 등장 여부를 외생적 설명 변수로 간주하였으나, 실제로는 평점과 동시적으로 형성될 가능성이 존재하므로 엄밀한 인과 추정으로 해석하기에는 한계가 있다.


향후 연구에서는 리뷰 데이터와 행동 로그 데이터를 결합한 다층 분석, 복수 서비스 간 경쟁 네트워크 분석, 기능 도입 전후의 차이-차이(DID) 분석, LLM 기반 자동 키워드 추출과 사전 기반 접근의 비교 등을 통해 기능 경쟁력 개념의 외적 타당성과 설명력을 더욱 확장할 수 있을 것이다.



제 2 장 선행연구 고찰


1. 사용자 생성 콘텐츠(UGC)와 온라인 리뷰의 정보적 가치
사용자 생성 콘텐츠(User-Generated Content, UGC)는 이용자가 자발적으로 생성 하고 공유하는 텍스트, 평점, 후기 등으로 구성되며, 디지털 플랫폼 환경에서 고객 경험과 인식이 축적되는 중요한 데이터 자원이다. UGC는 단순한 의견 표출을 넘어 서비스에 대한 태도 형성, 재이용 의도, 관계 몰입 등 다양한 행동 결과와 연결될 수 있다는 점에서 학술적·실무적 관심이 지속되어 왔다.[김희연, 윤성준. (2023)]
온라인 리뷰는 UGC의 대표적 형태로, 평점(정량)과 텍스트(정성)가 결합된 구조를 가진다. 이 결합 구조는 만족과 불만 수준을 수치로 확인하는 데 그치지 않고, 평가가 형성된 맥락과 이유를 함께 해석할 수 있게 한다. 선행연구는 리뷰 텍스트의 주제나 내용 유형이 리뷰 유용성 인식에 영향을 미칠 수 있음을 보고하며, 텍스트 자체가 중요한 분석 대상으로 기능함을 보여준다.[김형준, 황용석. (2023)]
특히 앱 마켓 리뷰는 동일 카테고리 내 복수 서비스에 대해 유사한 형식(평점, 텍스트, 작성 시점)으로 축적된다는 점에서, 단일 서비스 진단을 넘어 경쟁 비교 연구를 수행할 수 있는 공통 관측 프레임을 제공한다. 이는 내부 행동 데이터가 비공개로 제한되는 현실에서, 경쟁 서비스 간 비교를 가능하게 하는 대체 데이터라는 점에서 의미가 있다.

2. 앱 마켓 리뷰 데이터의 특성과 분석상 고려사항
앱 마켓 리뷰는 서비스 이용 이후 작성되는 사후 평가 데이터이면서, 사용자가 겪은 문제 상황과 원인을 텍스트로 기술한다는 점에서 개선 단서가 풍부한 자료이다. 실제로 앱 리뷰 데이터를 활용해 만족과 불만 요인을 도출하고, 로지스틱 회귀 및 오즈비 분석으로 영향력을 비교한 연구가 보고된 바 있다.[김하람, 전찬희, 김동수. (2022)] 또한 OTT, 홈 애플리케이션 등 다양한 서비스 카테고리에서 텍스트 마이닝을 통해 이용 요인과 요구를 구조화한 연구가 축적되어 왔다[이성준. (2022)][조정희, 최준호. (2025)].
다만 앱 리뷰 데이터는 다음과 같은 한계가 존재한다. 
첫째, 리뷰는 자발적 참여 기반이므로 표본 선택 편향(self-selection bias)이 발생할 수 있다. 둘째, 업데이트와 이벤트 그리고 장애 등 특정 시점의 외생적 충격이 단기적으로 반영될 수 있다. 셋째, 리뷰는 인식(perception) 기반 데이터이므로 실제 행동 지표(전환율, 결제 성공률 등)와 직접적으로 일치한다고 단정하기 어렵다. 넷째, 동일 기능이라도 표현 방식(동의어·은어·축약어)이 다양하여 텍스트 정제와 키워드 정의의 일관성이 요구된다.
따라서 리뷰 기반 연구는 데이터 수집 범위와 기간을 명확히 하고, 텍스트 전처리 및 키워드 정의 규칙을 재현 가능하게 제시하며, 결과 해석 시 데이터 구조적 한계를 함께 논의할 필요가 있다.

3. 리뷰 텍스트 마이닝 접근의 유형
리뷰 텍스트 분석은 크게 (1) 주제 도출 중심의 탐색적 접근, (2) 예측 중심의 분류·모형화 접근, (3) 해석 중심의 설명적 접근으로 구분할 수 있다. 
첫째, LDA 등 토픽 모델링은 리뷰에 내재된 주제 구조를 탐색하는 데 유용하며, 서비스의 주요 만족과 불만 요인을 요약하고 구조화하는 데 강점이 있다.[이성준. (2022)] 
둘째, 임베딩·딥러닝 기반 접근은 표현력과 예측 성능 측면에서 강점을 갖지만, 결과 해석이 제한적일 수 있으며 “어떤 기능을 왜 개선해야 하는가”라는 의사결정 질문에 직접 연결되기 어렵다는 한계가 지적된다[구보림, 황원일. (2025)][이미선, 안현철. (2023).].
 셋째, 로지스틱 회귀와 같은 통계 모형 기반 접근은 변수의 영향 방향과 크기를 해석 가능한 형태로 제시할 수 있다는 점에서 의사결정 친화적 특성을 가진다. 특히 선행연구는 리뷰 키워드를 설명변수로 두고 긍정/부정 평점에 대한 영향력을 추정한 뒤, 오즈비를 통해 요인별 영향력을 비교하는 방식을 제시한 바 있다[김하람, 전찬희, 김동수. (2022)].


4. 로지스틱 회귀와 오즈비의 해석적 의의
로지스틱 회귀는 종속변수가 이분형(예: 긍정/부정)일 때, 설명변수가 결과 발생 확률에 미치는 영향을 추정하는 대표적 모형이다. 오즈비(Odds Ratio)는 해당 변수의 영향력을 직관적인 크기 비교가 가능한 지표로 변환해 준다는 점에서, 기능 단위 개선 포인트를 정량적으로 설명하는 데 활용될 수 있다[김하람, 전찬희, 김동수. (2022)].
본 연구는 이 해석 가능성을 확장하여, 동일 기능 키워드의 영향력을 서비스 간 비교함으로써 기능 경쟁력을 “조건부 영향력의 차이”로 정의한다는 점에서 차별성을 가진다.


5. 선행연구와 본 연구와의 차별성
선행연구는 리뷰 데이터로 만족·불만 요인을 도출하거나[이성준. (2022)], 경쟁 앱 비교 분석을 수행하거나[김하람, 전찬희, 김동수. (2022)], 텍스트 마이닝 기법으로 의미 구조를 분석하는 연구[구보림, 황원일. (2025)][이미선, 안현철. (2023).]로 축적되어 왔다. 그러나 다음과 같은 공백이 존재한다.
첫째, 동일 기능을 기준으로 서비스 간 기능 경쟁력을 정량적으로 정의하는 틀이 충분히 정립되지 않았다. 둘째, 영향력 비교 결과를 기능 개선 우선순위 규칙으로 연결하는 절차가 체계화되어 있지 않다. 셋째, 특정 카테고리에 국한되지 않고 타 서비스 도메인으로 확장 가능한 프레임워크 제시가 제한적이다.
본 연구는 (1) 기능 키워드 영향력 추정, (2) 동일 기능에 대한 서비스 간 영향력 격차 산출, (3) 격차 기반 우선순위 도출 규칙 제시라는 연결 구조를 통해 리뷰 분석을 진단적 분석에서 의사결정 지원 프레임워크로 확장한다.


6. 정리
본 장에서는 UGC와 온라인 리뷰의 정보적 가치, 앱 마켓 리뷰 데이터의 특성과 한계, 리뷰 텍스트 분석 접근 유형, 로지스틱 회귀 및 오즈비의 해석적 의미를 검토하였다. 이를 통해 기능 단위 경쟁력을 서비스 간 비교로 정량화하고 우선순위 규칙으로 연결하려는 본 연구의 필요성과 연구 공백을 확인하였다. 다음 장에서는 이를 실증적으로 검증하기 위한 데이터 수집, 변수 정의, 모형 설정 및 분석 절차를 구체적으로 제시한다.

제 3 장 연구 방법론


1. 연구 설계
본 연구는 앱 마켓 리뷰 텍스트에 포함된 기능 관련 키워드가 평점(긍·부정)에 미치는 영향을 추정하고, 동일 기능 키워드의 영향력을 서비스 간 비교함으로써 기능 경쟁력을 정량화하는 비교 분석 연구이다. 연구 목적은 기능 경쟁력을 “동일 기능이 언급되었을 때 긍·부정 평가로 이어질 가능성의 차이”로 개념화하고, 이 차이를 기능 개선 우선순위 결정에 활용 가능한 규칙으로 전환하는 데 있다.
연구 절차는 (1) 리뷰 데이터 수집, (2) 텍스트 전처리 및 기능 키워드 정의, (3) 평점 이분화, (4) 로지스틱 회귀 추정, (5) 오즈비 산출, (6) 서비스 간 영향력 격차 검정 및 산출, (7) 우선순위 규칙 적용 및 강건성 검증으로 구성된다.

2.2 데이터 수집 범위
본 연구는 비교 가능한 공개 리뷰 데이터를 활용해 기능 단위 경쟁력을 추정하는 것을 목표로 하므로, 동일한 기준과 형식으로 수집 가능한 앱 마켓을 분석 대상으로 설정하였다. 이에 따라 리뷰 수집 플랫폼은 Google Play Store로 한정하였다. iOS App Store의 경우에도 리뷰가 공개되어 있으나, (1) 국가 · 계정 · OS 버전에 따른 노출/정렬 정책 차이, (2) 리뷰 접근 및 수집 방식의 제약(조회 제한 등)으로 인해 동일 기간·동일 조건의 표본을 안정적으로 확보하기 어렵고, (3) 결과적으로 서비스 간 비교에서 표본 구성 차이가 분석 결과에 개입할 위험이 존재한다. 반면 Google Play Store는 리뷰 텍스트, 평점, 작성일 등 핵심 항목이 비교적 일관된 형태로 제공되어, 동일 기준으로 두 서비스의 리뷰를 수집·정제하고 재현 가능한 분석을 수행하는 데 적합하다. 따라서 본 연구는 데이터 수집의 일관성과 비교 가능성을 확보하기 위해 iOS 마켓을 제외하고 Android 마켓 데이터로 분석 범위를 설정하였다.
수집 기간은 “최근 1년”으로 한정하였다. 리뷰 데이터는 앱 업데이트, 정책 변경, 장애 발생 등 특정 시점의 사건에 의해 단기적으로 감정이 과대표집될 수 있으며, 반대로 너무 긴 기간을 포함하면 과거 버전에서 발생한 문제(이미 개선된 이슈)가 현재 기능 경쟁력으로 오인될 가능성이 있다. 본 연구는 기능 경쟁력을 현시점의 사용자 경험과 가까운 범위에서 추정하는 것을 목적으로 하므로, (1) 비교 대상 서비스의 기능 구성과 시장 환경이 상대적으로 유사하게 유지되는 구간을 확보하고, (2) 계절성·이벤트성 변동을 1년 단위로 포괄하여 극단적 단기 충격을 완화하며, (3) 분석 결과가 “현재 개선 우선순위”로 해석 가능하도록 시간적 타당성을 확보하기 위해 최근 1년을 수집 기간으로 설정하였다. 또한 업데이트 직후 등 특정 시점 효과가 분석 결과를 왜곡할 가능성에 대비하여, 작성 월 더미 등 시점 통제 변수를 포함하고 기간 분할 민감도 분석을 통해 결과의 안정성을 추가로 검증한다.

3. 변수 정의
3.1 종속 변수 : 긍/부정 평가 점수
본 연구의 종속 변수는 리뷰 평점의 긍·부정 여부이다. 앱 마켓에서 제공되는 1점에서 5점까지의 평점 중 4점과 5점은 긍정 평가, 1점과 2점은 부정 평가로 구분하여 이분형 변수로 구성하였다. 이는 사용자가 명확한 만족 또는 불만 의사를 표현한 구간을 중심으로 기능 영향력을 추정하기 위함이다.
3점 평점은 긍·부정이 명확하지 않은 중립적 평가로 간주하여 기본 분석에서는 제외하였으며, 평점 기준 변화에 따른 결과의 안정성을 확인하기 위해 강건성 검증 단계에서 포함 여부를 달리하여 민감도 분석을 수행한다.

3.2 독립 변수 : 기능 키워드
독립 변수는 리뷰 텍스트에 특정 기능 또는 경험 요소가 언급되었는지 여부이다. 기능 키워드는 결제, 인증, 송금, 오류, 속도 등 서비스의 핵심 기능과 직접적으로 관련된 표현을 중심으로 정의하며, 텍스트 전처리 과정을 거쳐 구축된 기능 키워드 사전을 기준으로 각 리뷰에 대해 등장 여부를 판별한다.
리뷰 i에서 기능 키워드 k가 한 번 이상 등장하면 1, 등장하지 않으면 0으로 처리하는 더미 변수 형태로 구성한다. 이를 통해 “해당 기능이 언급되었을 때 평점이 어떻게 달라지는가”를 조건부 확률의 변화로 추정할 수 있도록 설계하였다.

3.3 통제 변수
기능 키워드의 효과를 보다 정확하게 추정하기 위해, 평점에 영향을 줄 수 있는 잠재적 교란 요인을 통제 변수로 포함한다.
첫째, 리뷰 길이(문자 수 또는 토큰 수)는 장문의 리뷰가 보다 구체적 불만이나 상세 경험을 포함할 가능성이 있다는 점을 고려하여 포함하였다. 
둘째, 작성 월 변수는 특정 시점의 외생적 사건(업데이트, 정책 변경, 프로모션 등)에 따른 일시적 평점 변동을 통제하기 위함이다. 
셋째, 대규모 업데이트 직후 여부는 기술적 장애나 구조적 개편이 단기적으로 부정 리뷰를 증가시킬 가능성을 고려하여 데이터 확보가 가능한 경우에 포함한다.
통제 변수의 산출 기준과 더미 구성 방식은 두 서비스에 동일하게 적용하여 비교 가능성을 확보한다.

4. 텍스트 전처리 및 기능 키워드 구성
리뷰 텍스트는 사용자 자유 서술 형태이므로 분석을 위해 정규화, 토큰화, 불용어 제거 등의 전처리를 수행한다. 또한 동일 기능을 지칭하는 다양한 표현을 통합하기 위해 동의어 및 유사 표현을 사전에 정의하고 기능 키워드 사전을 구축한다.
​​기능 키워드 사전의 구성은 연구자의 임의적 판단에 의존하지 않도록, 데이터 기반의 빈도 분석을 선행한 후 단계적으로 구축하였다. 구체적으로, 전처리 완료된 전체 리뷰 텍스트를 대상으로 형태소 단위의 명사 및 핵심 동사를 추출하고, 상위 빈도 단어 목록을 도출하였다. 이후 상위 빈도 키워드 중 서비스의 핵심 기능과 직접적으로 연관된 표현을 1차 후보군으로 선정하였다.
선정 기준은 다음과 같다. 첫째, 단순 감탄사나 일반 감성 표현(예: “좋아요”, “별로”, “최고”)은 제외하였다. 둘째, UI와 정책 그리고 기술적 기능과 직접적으로 연결되는 표현(예: 결제, 인증, 송금, 오류, 업데이트, 속도 등)을 우선 포함하였다. 셋째, 동일 기능을 지칭하는 복수 표현(예: “결제 안됨”, “결제가 안 돼요”, “결제 오류”)은 동의어 통합 규칙에 따라 하나의 기능 키워드로 통합하였다.
이와 같은 절차를 통해 데이터 기반으로 도출된 상위 빈도 기능 관련 키워드를 중심으로 기능 사전을 구축하였으며, 최종 사전은 빈도 기준과 도메인 적합성 검토를 모두 충족한 항목으로 구성하였다. 이를 통해 기능 키워드 정의의 객관성과 재현 가능성을 확보하고자 하였다.
또한 두 서비스 간 공통적으로 상위 빈도에 등장하는 기능 키워드를 중심으로 비교 분석을 수행하여, 특정 서비스에만 특이적으로 나타나는 표현이 분석 결과에 과도하게 영향을 미치는 것을 방지하였다.

5. 로지스틱 회귀모형 추정
본 단계의 목적은 “특정 기능이 리뷰에서 언급되었을 때, 해당 리뷰가 긍정 평가로 이어질 확률이 얼마나 달라지는가?”를 통계적으로 추정하는 것이다. 로
지스틱 회귀는 결과가 ‘긍정/부정’처럼 두 가지로 나뉘는 경우에, 설명변수가 결과 발생 확률에 미치는 영향을 추정하는 대표적 모형이다. 확률값이 0과 1 사이에 제한되는 특성을 고려하여 확률 자체가 아닌 ‘오즈(odds)’의 로그값(log-odds)을 선형식으로 모형화한다.

6. 서비스 간 영향력 격차 검정 및 OR 산출
동일 기능 키워드의 영향력이 서비스에 따라 달라지는지 검정하기 위해, 기능 키워드와 서비스 더미의 상호작용항을 포함한 모형을 추가로 추정한다. 상호작용항의 유의성은 동일 기능의 영향력이 서비스 간 차이를 보이는지에 대한 통계적 근거로 활용된다.
 
여기서 는 서비스 더미 변수(A=1, B=0)이며, 는 ‘서비스 자체의 기본 수준 차이(기능과 무관한 평균적 차이)’를 통제하기 위한 계수이다. 
는 기능 의 효과가 서비스에 따라 얼마나 달라지는지를 나타내는 상호작용 계수로, 가 통계적으로 유의하면 동일 기능 키워드의 영향력이 서비스 간 유의하게 다름을 의미한다.
상호작용항의 유의성 검정이 ‘차이가 존재하는지’를 판단하는 절차라면, ΔOR은 그 차이의 방향과 크기를 정량적으로 보여주는 지표이다. 서비스 간 동일 기능의 영향력 차이를 ‘격차 지표’로 정량화하기 위해 다음과 같이 오즈비 격차를 정의한다.

7. 기능 개선 우선순위 도출 규칙
본 연구는 기능 경쟁력 격차를 단순 비교에서 끝내지 않고, 실제 개선 의사결정에 활용 가능한 우선순위 규칙으로 전환하는 것을 목표로 한다. 이를 위해 서비스 간 영향력 격차(OR)와 자사 취약도(부정 평가 관련 지표)를 결합하여 우선순위를 산출한다. 즉, 경쟁 서비스 대비 열위가 크고 동시에 부정 평가와 연관이 큰 기능을 상위 개선 대상으로 도출하는 구조이다. 우선순위 산출 규칙은 재현 가능하도록 명시적으로 제시한다.

9. 분석의 식별 가정
본 연구는 기능 키워드의 등장 여부가 평점에 미치는 조건부 영향력을 추정하기 위해 다음과 같은 가정을 전제한다.
첫째, 통제변수를 포함한 조건 하에서 기능 키워드 등장 여부와 오차항 간 상관이 없다는 조건부 독립성 가정이다.
둘째, 기능 키워드는 평점에 대한 인식적 평가를 반영하는 지표로 해석되며, 인과적 효과라기보다는 조건부 연관성(association)으로 해석한다.
셋째, 서비스 간 비교는 동일 기간·동일 플랫폼 표본을 전제로 하여 수행되며, 표본 구성 차이에 의한 체계적 왜곡이 최소화되었다고 가정한다.
이러한 가정 하에서 추정된 결과는 기능 단위 경쟁력의 상대적 비교 지표로 활용된다.
